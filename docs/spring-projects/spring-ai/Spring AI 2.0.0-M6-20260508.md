Spring AI 2.0.0-M6 Release Notes

🎯 Highlights

This release includes 7 new features, 18 bug fixes, 5 documentation improvements, 30 other improvements.

⏪ Breaking Changes

* PromptChatMemoryAdvisor has been removed. Chat memory advisors now require an explicit conversation ID to be provided, improving predictability and eliminating ambiguous implicit state management. Applications using PromptChatMemoryAdvisor or relying on implicit conversation IDs must be updated. <a href="https://github.com/spring-projects/spring-ai/commit/c3c7c86d48e2606c3d04f48ad35e96ffd877b307">c3c7c86</a>, <a href="https://github.com/spring-projects/spring-ai/commit/59ab7521f0a8f67c89359e910a20472d572b4dd9">59ab752</a>
* The class OpenAiConnectionProperties has been renamed to OpenAiCommonProperties to better reflect its purpose as shared configuration across OpenAI integrations. <a href="https://github.com/spring-projects/spring-ai/commit/01400e27232e6fcc9dd57a6e321a5c598dbff07e">01400e2</a>
* OpenAI properties classes (e.g., OpenAiChatProperties, OpenAiEmbeddingProperties) no longer extend AbstractOpenAiOptions. This changes the class hierarchy and may affect code that relied on the options being available directly on properties classes. <a href="https://github.com/spring-projects/spring-ai/commit/7b2b4367c03069939e824be0cd8292636b7dda9f">7b2b436</a>
* Setter methods have been removed from OpenAI options classes in favor of the builder pattern. Code that called setters directly on options (e.g., OpenAiChatOptions) will need to be updated to use the builder API. <a href="https://github.com/spring-projects/spring-ai/commit/f119b2f72e6cc2e6ee3e8a9f3971461e1f5ab8be">f119b2f</a>
* The @NestedConfigurationProperty annotations have been removed from OpenAI options fields. This may affect IDE auto-completion and Spring Boot configuration metadata for nested OpenAI properties. <a href="https://github.com/spring-projects/spring-ai/commit/5f82cdd44445c9f1a02c619e9118e3b34e0806d5">5f82cdd</a>
* The SAP HANA DB vector store module has been removed from Spring AI. Users relying on this integration must migrate to an alternative vector store provider. <a href="https://github.com/spring-projects/spring-ai/commit/35b659e3fee17668396a95c9fb1cb1aab0658d9e">35b659e</a>
* The Infinispan vector store module has been removed from Spring AI. Users relying on this integration must migrate to an alternative vector store provider. <a href="https://github.com/spring-projects/spring-ai/commit/195d418f91a89f991bef58b05a9250b95fa776c4">195d418</a>
* Mutable setter methods have been removed from Mistral AI options classes. Configuration must now be done through constructors or builders, enforcing immutability. <a href="https://github.com/spring-projects/spring-ai/commit/06015b7d3f72bcbc621342f8ee4f27b55adc3b91">06015b7</a>
* Mutable setter methods have been removed from MiniMax GenAI options classes. Configuration must now be done through constructors or builders, enforcing immutability. <a href="https://github.com/spring-projects/spring-ai/commit/222bc7ddfa1870a9bdcd1b0fb4cb7b25f1a9b02f">222bc7d</a>
* Mutable setter methods have been removed from Google GenAI options classes. Configuration must now be done through constructors or builders, enforcing immutability. <a href="https://github.com/spring-projects/spring-ai/commit/047f9a8e2155db1ea2c50bb1df72bcb4ff9c4728">047f9a8</a>
* Mutable setter methods have been removed from ElevenLabs options classes. Configuration must now be done through constructors or builders, enforcing immutability. <a href="https://github.com/spring-projects/spring-ai/commit/304fe1be122ac2eb1589ef9057467b519969813a">304fe1b</a>
* Mutable setter methods have been removed from Bedrock options classes. Users must now use the immutable builder pattern or constructor-based initialization for Bedrock model options. <a href="https://github.com/spring-projects/spring-ai/commit/2b0b45b44556068e73e26e8527514cd9c1d6b785">2b0b45b</a>
* Mutable setter methods have been removed from DeepSeek options classes. Users must now use the immutable builder pattern or constructor-based initialization for DeepSeek model options. <a href="https://github.com/spring-projects/spring-ai/commit/a4e99c5a6c6b0710d975004211551389454b0186">a4e99c5</a>
* Anthropic configuration properties have been refactored and mutable setter methods removed from Anthropic options classes. Users must update to use the new property structure and immutable builder patterns. <a href="https://github.com/spring-projects/spring-ai/commit/556b5f22fc2d4ad82c08becfc86aa7f463da0352">556b5f2</a>
* Several utility methods have been removed from the ModelOptionsUtils class. Users relying on these methods need to update their code to use alternative approaches. <a href="https://github.com/spring-projects/spring-ai/pull/5941" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5941/hovercard">#5941</a>
* Unnecessary Jackson annotations have been removed from OpenAiChatOptions. This may affect custom serialization/deserialization if users relied on these annotations. <a href="https://github.com/spring-projects/spring-ai/commit/6ed5928c1a8e3d8d2799357180f8e813163066fa">6ed5928</a>
* Setter methods have been removed from OllamaChatOptions as part of an API cleanup. Users must now configure options using the builder pattern instead of calling setters directly. <a href="https://github.com/spring-projects/spring-ai/pull/5923" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5923/hovercard">#5923</a>

:warning: Upgrading Notes

* PromptChatMemoryAdvisor has been removed. Replace usages with the appropriate alternative chat memory advisor implementation and ensure all chat memory advisors are configured with an explicit conversation ID rather than relying on implicit identification. <a href="https://github.com/spring-projects/spring-ai/commit/c3c7c86d48e2606c3d04f48ad35e96ffd877b307">c3c7c86</a>, <a href="https://github.com/spring-projects/spring-ai/commit/59ab7521f0a8f67c89359e910a20472d572b4dd9">59ab752</a>
* OpenAiConnectionProperties is now OpenAiCommonProperties. OpenAi*Properties classes no longer extend AbstractOpenAiOptions — options are now configured separately. Setters have been removed; use the builder pattern instead (e.g., OpenAiChatOptions.builder()...build()). Review upgrade-notes.adoc for detailed migration guidance. <a href="https://github.com/spring-projects/spring-ai/commit/52c952ecb3e4c024cb6a8aff9a3fd48cc3dc302f">52c952e</a>
* If you use Tanzu bindings, the TanzuBindingsPropertiesProcessor has been upgraded to work with the new OpenAI configuration properties structure. Ensure your Tanzu bindings configuration is updated accordingly. <a href="https://github.com/spring-projects/spring-ai/commit/d27f6fd9e63f94eab8a07daa054f9a51b83c354c">d27f6fd</a>
* If you were using setter methods on PostgresMlEmbeddingOptions, StabilityAiImageOptions, or calling options.setInternalToolExecutionEnabled() / options.setOutputSchema() on ChatOptions, you must migrate to the builder pattern. Use the respective builder class (e.g., PostgresMlEmbeddingOptions.builder().build()) to construct options immutably. <a href="https://github.com/spring-projects/spring-ai/commit/a0ad1c76f8827392ce702c1db2f4b8681becb506">a0ad1c7</a>, <a href="https://github.com/spring-projects/spring-ai/commit/741a6cc9e923096416c7ca61f2c0f60bc2cc95e8">741a6cc</a>, <a href="https://github.com/spring-projects/spring-ai/pull/5957" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5957/hovercard">#5957</a>
* If using the SAP HANA DB or Infinispan vector store integrations, you must migrate to an alternative supported vector store (e.g., pgvector, Redis, Chroma, Pinecone, etc.). Remove the corresponding starter dependencies from your build configuration. <a href="https://github.com/spring-projects/spring-ai/commit/35b659e3fee17668396a95c9fb1cb1aab0658d9e">35b659e</a>
* Setter methods have been removed from Mistral AI, MiniMax GenAI, Google GenAI, and ElevenLabs options classes. Update your code to use the builder pattern or constructor-based configuration instead of calling setter methods directly. <a href="https://github.com/spring-projects/spring-ai/commit/06015b7d3f72bcbc621342f8ee4f27b55adc3b91">06015b7</a>
* With setters removed from Bedrock, DeepSeek, and Anthropic options classes, users must migrate to the builder pattern. Replace calls like <code>options.setTemperature(0.7)</code> with the equivalent builder method: <code>Options.builder().temperature(0.7).build()</code>. Review Anthropic property names as they have been refactored. <a href="https://github.com/spring-projects/spring-ai/commit/2b0b45b44556068e73e26e8527514cd9c1d6b785">2b0b45b</a>
* If your application directly calls any of the removed ModelOptionsUtils methods, you will need to update your code. Review the PR for details on which methods were removed and what alternatives are available. <a href="https://github.com/spring-projects/spring-ai/pull/5941" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5941/hovercard">#5941</a>
* With setters removed from OllamaChatOptions, users must update their code to use the builder API. Replace any direct setter calls (e.g., options.setModel(...)) with the equivalent builder methods (e.g., OllamaChatOptions.builder().model(...).build()). <a href="https://github.com/spring-projects/spring-ai/pull/5923" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5923/hovercard">#5923</a>
* If you were previously using string values for OpenAiEmbeddingOptions#encodingFormat, update your code to use the new enum type. This provides compile-time safety and ensures only valid encoding formats are used. <a href="https://github.com/spring-projects/spring-ai/commit/de1824ffed91ef6103a6860e77993f6c838402bf">de1824f</a>

⭐ New Features

* Enhanced observability support for chat models, providing better monitoring and tracing capabilities for AI model interactions. <a href="https://github.com/spring-projects/spring-ai/commit/c3bdbb04a4d51344339d4fb5b700bbb81331fc5f">c3bdbb0</a>
* OpenAI options classes now use an improved builder pattern, providing a cleaner and more consistent API for constructing option objects programmatically. <a href="https://github.com/spring-projects/spring-ai/commit/5568c7b498f1e9d61b472ad4b737702031272134">5568c7b</a>
