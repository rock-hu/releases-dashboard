Spring AI 2.0.0-M3 Release Notes

🎯 Highlights

This release includes 23 new features, 45 bug fixes, 16 documentation improvements, 67 other improvements.

📢 Noteworthy

* MCP Annotations are now moved from spring-ai-community/mcp-annotations (<code>org.springaicommunity.mcp</code>) into Spring AI core (<code>org.springframework.ai.mcp.annotation</code>). <a href="https://github.com/spring-projects/spring-ai/commit/585d618cc88dad890612673882f999438cc8688a">585d618</a>
* Spring-specific MCP transport implementations are now moved from the MCP Java SDK (<code>io.modelcontextprotocol.sdk</code>) into the Spring AI project (<code>org.springframework.ai.mcp</code>). <a href="https://github.com/spring-projects/spring-ai/commit/0045513db7b929d5131d0c7fa60a83a77b525634">0045513</a>
* The OpenAI SDK integration now uses the new builder pattern for <code>OpenAiSdkChatOptions</code>, providing a more fluent API for configuration. <a href="https://github.com/spring-projects/spring-ai/commit/3508cc10caa0853908160d652bda79b12ce1d581">3508cc1</a>
* A deprecated shim has been added for the <code>disableMemory()</code> method. Users should migrate to <code>disableInternalConversationHistory()</code> as the preferred API <a href="https://github.com/spring-projects/spring-ai/commit/9893de661735e6ab81bf9c7e1f6a4fabf80d404e">9893de6</a>
* The <code>OpenAiChatModel#internalCall</code> method and other internal methods have had their access control scope optimized to private, improving code encapsulation and reducing unintended usage of internal APIs. <a href="https://github.com/spring-projects/spring-ai/commit/e86b8d720f3b6f9b30a41f92b2e078dcc99c157a">e86b8d7</a>
* Reverted a breaking API change that made <code>ModelRequest#getOptions</code> non-null, restoring backward compatibility <a href="https://github.com/spring-projects/spring-ai/pull/5403" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5403/hovercard">#5403</a>
* Simplified tool context management by removing automatic tool call history inclusion <a href="https://github.com/spring-projects/spring-ai/commit/8ab7f7ea8f205e0d7d775dc82194a7e50f9038ec">8ab7f7e</a>

:warning: Breaking Changes <strong>Breaking Changes:</strong> 2.0.0-M3 includes several breaking changes, including MCP annotation package renames, MCP transport artifact relocations, Jackson 2 → Jackson 3 migration, and removal of conversation history from <code>ToolContext</code>. Review the <a href="https://docs.spring.io/spring-ai/reference/2.0-SNAPSHOT/upgrade-notes.html#upgrading-to-2-0-0-M3" rel="nofollow">Upgrade Notes</a> before upgrading.

* MCP Annotations are now moved from spring-ai-community/mcp-annotations (<code>org.springaicommunity.mcp</code>) into Spring AI core (<code>org.springframework.ai.mcp.annotation</code>). <a href="https://github.com/spring-projects/spring-ai/commit/585d618cc88dad890612673882f999438cc8688a">585d618</a>. Follow the <a href="https://docs.spring.io/spring-ai/reference/2.0-SNAPSHOT/upgrade-notes.html#_mcp_annotations_migrated_into_spring_ai" rel="nofollow">migration instructions</a>.
* <code>McpAsyncClientCustomizer</code> and <code>McpSyncClientCustomizer</code> have been removed and replaced by a single generic interface <code>McpClientCustomizer&lt;B&gt;</code>. <a href="https://github.com/spring-projects/spring-ai/commit/272c7361287d4511ed57648705c7a12c6c3d0df9">272c736</a>. Follow the <a href="https://docs.spring.io/spring-ai/reference/2.0-SNAPSHOT/upgrade-notes.html#_mcp_client_customizer_api_consolidated" rel="nofollow">migration instructions</a>.
* Spring-specific MCP transport implementations are now moved from the MCP Java SDK (<code>io.modelcontextprotocol.sdk</code>) into the Spring AI project (<code>org.springframework.ai.mcp</code>). <a href="https://github.com/spring-projects/spring-ai/commit/0045513db7b929d5131d0c7fa60a83a77b525634">0045513</a>. Follow the <a href="https://docs.spring.io/spring-ai/reference/2.0-SNAPSHOT/upgrade-notes.html#_mcp_spring_transport_modules_moved_to_spring_ai" rel="nofollow">migration instructions</a>.
* Spring AI now uses Jackson 3 (<code>tools.jackson</code> package) instead of Jackson 2 (<code>com.fasterxml.jackson</code> package) <a href="https://github.com/spring-projects/spring-ai/commit/42043bc7d99487bb8dcec2713393cca5b7daed93">42043bc</a> which is used by default in Spring Boot 4, if you were using previous milestone make sure to refine your Jackson dependencies accordingly.
* Conversation History Removed from <code>ToolContext</code> <a href="https://docs.spring.io/spring-ai/reference/2.0-SNAPSHOT/upgrade-notes.html#_conversation_history_removed_from_toolcontext" rel="nofollow">Migration Instructions</a>
* Builders exposed to users no longer require explicit generics, simplifying the API surface. This affects <code>MiniMaxChatOptions</code> and potentially other model provider option builders. <a href="https://github.com/spring-projects/spring-ai/commit/7470fd8c98ccf5566cd33e49f3c36d78ff5f6c8b">7470fd8</a>. Code using explicit generics should be updated to use simplified builder syntax.
* The Anthropic integration has been replaced from a RestClient-based implementation to use the official Anthropic Java SDK, which may require code changes for applications using the Anthropic model provider. <a href="https://github.com/spring-projects/spring-ai/commit/b0ce25aed319d4fb0c80535375a81d8ce57bdb80">b0ce25a</a>. If you are using the Anthropic model provider, review your code as the internal implementation has changed from <code>RestClient</code> to the official Anthropic Java SDK. The public API should remain compatible, but custom configurations may need adjustment.
* The <code>disableMemory()</code> method has been renamed to <code>disableInternalConversationHistory()</code> for better clarity about its purpose <a href="https://github.com/spring-projects/spring-ai/commit/657252ae3be5e7d97e45319a838b88de949a2f0d">657252a</a>. Replace calls to <code>disableMemory()</code> with <code>disableInternalConversationHistory()</code>. A deprecated shim is provided for backward compatibility.
* Huggingface model and autoconfiguration have been moved out of the main Spring AI repository <a href="https://github.com/spring-projects/spring-ai/commit/f21b082ac18758003924786a4ed082f7555f39af">f21b082</a>
* <code>DeepSeekChatOptions</code> has been updated to use the modern builder pattern for consistency with other chat model options in Spring AI. Existing code using the old constructor pattern will need to be updated. <a href="https://github.com/spring-projects/spring-ai/pull/5552" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5552/hovercard">#5552</a>. Update your code to use the new builder pattern instead of direct constructor calls. Example: <code>DeepSeekChatOptions.builder().temperature(0.7).build()</code>
* <code>BedrockChatOptions</code> has been updated to use the modern builder pattern for consistency with other chat model options in Spring AI. Existing code using the old constructor pattern will need to be updated. <a href="https://github.com/spring-projects/spring-ai/pull/5550" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5550/hovercard">#5550</a>
* <code>MistralAiChatOptions</code> now uses builder pattern for configuration, replacing direct constructor/setter usage <a href="https://github.com/spring-projects/spring-ai/pull/5551" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5551/hovercard">#5551</a>. Update code using <code>MistralAiChatOptions</code> to use the new builder pattern instead of constructors or setters.
* <code>AzureOpenAiChatOptions</code> now uses builder pattern for configuration, replacing direct constructor/setter usage <a href="https://github.com/spring-projects/spring-ai/pull/5544" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5544/hovercard">#5544</a>. Update code using <code>AzureOpenAiChatOptions</code> to use the new builder pattern instead of constructors or setters.
* <code>AnthropicChatOptions</code> now uses builder pattern for configuration, replacing direct constructor/setter usage <a href="https://github.com/spring-projects/spring-ai/pull/5541" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5541/hovercard">#5541</a>. Update code using <code>AnthropicChatOptions</code> to use the new builder pattern instead of constructors or setters.
* Refactored <code>ChatOptions</code> builders to introduce builder merging pattern, which may affect custom implementations extending <code>ChatOptions</code> builders <a href="https://github.com/spring-projects/spring-ai/commit/3db5c2bd4b1047df98e6c38a4d3f2a2930b903e8">3db5c2b</a>. If you have custom implementations extending ChatOptions builders, review the new builder merging pattern and update your code accordingly. The new pattern is demonstrated in the <code>OllamaChatOptions</code> implementation.
* Claude 3 Opus, Sonnet, and Haiku models have been removed. Applications must migrate to Claude 4.x model variants. <a href="https://github.com/spring-projects/spring-ai/commit/628b020041f94fb223ffd873e94c9304c42bc10f">628b020</a>
* Internal methods in model implementations have been changed from protected/package-private to private access, improving encapsulation. This affects OpenAI and other model provider implementations. If you were extending these classes and accessing internal methods, you'll need to use public APIs instead. <a href="https://github.com/spring-projects/spring-ai/commit/da3bec4a8ab88e259be9827f7e898a5e699195ff">da3bec4</a>
* The <code>ModelRequest#getOptions</code> method now returns a non-null value. Code expecting null return values will need to be updated. <a href="https://github.com/spring-projects/spring-ai/commit/e4a4723c78f92a0bdb4ca5181d396576cc770165">e4a4723</a>. If your code checks for null returns from ModelRequest#getOptions, update it to handle the non-null guarantee. The method now always returns a valid options object.
* Previously deprecated constants in Mistral model enums have been removed. Update code to use the current model enum values. <a href="https://github.com/spring-projects/spring-ai/commit/96b408d6e56393cfaafd21c0025810dc4180b37e">96b408d</a>. If your code references deprecated Mistral model enum constants, update to use the current enum values as the deprecated constants have been removed.
* If your application uses Claude 3 Opus, Sonnet, or Haiku models, update model identifiers to their Claude 4.x equivalents (e.g., claude-sonnet-4-5-20250929). <a href="https://github.com/spring-projects/spring-ai/commit/628b020041f94fb223ffd873e94c9304c42bc10f">628b020</a>
* If you extended model implementation classes and accessed internal methods, verify that you're using public APIs only. Internal methods are now private and no longer accessible from subclasses. <a href="https://github.com/spring-projects/spring-ai/commit/da3bec4a8ab88e259be9827f7e898a5e699195ff">da3bec4</a>
