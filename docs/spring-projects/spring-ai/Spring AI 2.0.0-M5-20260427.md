Spring AI 2.0.0-M5 Release Notes

🎯 Highlights

This release includes 13 new features, 36 bug fixes, 14 documentation improvements, 41 other improvements.

⏪ Breaking Changes

* The MCP Java SDK has been upgraded to version 2.0.0-M2. This upgrade includes breaking API changes that require migration. Review the documented breaking changes before upgrading. <a href="https://github.com/spring-projects/spring-ai/pull/5879" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5879/hovercard">#5879</a>
* The <code>spring-ai-azure-openai</code> module has been removed from Spring AI. Azure OpenAI functionality is now available through the standard <code>spring-ai-openai</code> module, which provides full Azure OpenAI deployment support. Users must migrate their dependencies from <code>spring-ai-azure-openai</code> to <code>spring-ai-openai</code> and update their configuration accordingly. <a href="https://github.com/spring-projects/spring-ai/commit/3f5255f7125b90e770aa6660ae6a1e7756cb6548">3f5255f</a>
* The <code>ModelOptionsUtils.merge()</code> method has been removed from non-chat model implementations. Applications relying on this utility method for options merging in non-chat models (e.g., audio, embedding, image models) will need to update their code to handle options merging directly. <a href="https://github.com/spring-projects/spring-ai/pull/5789" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5789/hovercard">#5789</a>
* Several <code>ModelOptionsUtils.merge()</code> calls have been removed from chat model implementations as part of the options management refactoring. This may affect custom integrations that depend on the previous options merging behavior. <a href="https://github.com/spring-projects/spring-ai/pull/5766" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5766/hovercard">#5766</a>
* The MCP JSON dependency has been removed from the <code>spring-ai-client-chat</code> module. Projects that relied on this transitive dependency will need to add it explicitly if still required. <a href="https://github.com/spring-projects/spring-ai/pull/5773" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5773/hovercard">#5773</a> via <a href="https://github.com/spring-projects/spring-ai/issues/5760" data-hovercard-type="issue" data-hovercard-url="/spring-projects/spring-ai/issues/5760/hovercard">#5760</a>
* The <code>ChatClient</code> options merging behavior has been redesigned and moved to the <code>ChatClient</code> level via a new builder <code>combineWith()</code> method. This changes how model options are combined and may require updates to existing code that relies on the previous merging behavior. <a href="https://github.com/spring-projects/spring-ai/pull/5725" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5725/hovercard">#5725</a>
* The Vertex AI model and autoconfiguration modules for non-embedding use cases have been removed from Spring AI. Only the <code>spring-ai-vertex-ai-embedding</code> module remains. Users relying on Vertex AI chat or other non-embedding functionality must migrate to alternative model providers. Documentation has also been updated to reflect these removals. <a href="https://github.com/spring-projects/spring-ai/pull/5714" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5714/hovercard">#5714</a>
* The ZhipuAI model has been removed from the main Spring AI repository. Users currently using the ZhipuAI integration should check for alternative community-maintained modules or migrate to other supported model providers. <a href="https://github.com/spring-projects/spring-ai/pull/5700" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5700/hovercard">#5700</a>
* Oracle Cloud Infrastructure (OCI) GenAI support has been removed from the main Spring AI repository. Users relying on OCI GenAI will be able to migrate to a separate integration that will be released and documented shortly. <a href="https://github.com/spring-projects/spring-ai/pull/5695" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5695/hovercard">#5695</a>
* The <code>SpringAiTestAutoConfigurations</code> class has been removed. Projects that depend on this auto-configuration for testing will need to update their test setup accordingly. <a href="https://github.com/spring-projects/spring-ai/pull/5684" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5684/hovercard">#5684</a>

:warning: Upgrading Notes

* Users integrating with the Model Context Protocol (MCP) must review the breaking changes introduced in the 2.0.0-M2 SDK upgrade. Update your MCP-related code and configuration according to the migration documentation added in this release. <a href="https://github.com/spring-projects/spring-ai/pull/5879" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5879/hovercard">#5879</a>
* Users of the Anthropic module should review the newly added migration documentation to ensure their configurations and code are compatible with the latest module changes. <a href="https://github.com/spring-projects/spring-ai/pull/5881" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5881/hovercard">#5881</a>
* If you were using the <code>spring-ai-azure-openai</code> dependency, you must remove it and replace it with <code>spring-ai-openai</code>. Update your Spring configuration to use the OpenAI auto-configuration with Azure-specific properties (endpoint, API key, deployment name). The <code>spring-ai-openai</code> module now fully supports Azure OpenAI deployments. Refer to the updated documentation and release notes for the OpenAI Java SDK transition for detailed migration instructions. <a href="https://github.com/spring-projects/spring-ai/commit/3f5255f7125b90e770aa6660ae6a1e7756cb6548">3f5255f</a>
* The Pixtral 12B model has been removed and Pixtral Large is now deprecated. Update your configuration to use the currently recommended Mistral/Pixtral models to avoid disruptions. <a href="https://github.com/spring-projects/spring-ai/commit/d479ced6d6c8cde552bb876e76ed6716df5be5b0">d479ced</a>
* The <code>spring-ai-openai-sdk</code> module has been merged into <code>spring-ai-openai</code> and no longer exists as a separate artifact. If your project had a direct dependency on <code>spring-ai-openai-sdk</code>, remove it — all functionality is now provided by <code>spring-ai-openai</code> via the official <code>openai-java</code> SDK. No changes to existing spring.ai.openai.* configuration properties are required. <a href="https://github.com/spring-projects/spring-ai/pull/5779" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5779/hovercard">#5779</a>
* If your application or custom model integration calls <code>ModelOptionsUtils.merge()</code>, you will need to refactor to handle options merging without this utility. Review PRs <a href="https://github.com/spring-projects/spring-ai/pull/5789" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5789/hovercard">#5789</a> and <a href="https://github.com/spring-projects/spring-ai/pull/5766" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5766/hovercard">#5766</a> for guidance on the new options management approach. <a href="https://github.com/spring-projects/spring-ai/pull/5789" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5789/hovercard">#5789</a>
* If your project depends on MCP JSON functionality through the <code>spring-ai-client-chat</code> module, you must now declare this dependency explicitly in your build configuration following its removal as a transitive dependency. <a href="https://github.com/spring-projects/spring-ai/pull/5773" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5773/hovercard">#5773</a> via <a href="https://github.com/spring-projects/spring-ai/issues/5760" data-hovercard-type="issue" data-hovercard-url="/spring-projects/spring-ai/issues/5760/hovercard">#5760</a>
* If your application relied on the previous options merging behavior at the model level, you should migrate to using the new combineWith() builder method on ChatClient. Review the updated documentation for the new merging semantics. <a href="https://github.com/spring-projects/spring-ai/pull/5725" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5725/hovercard">#5725</a>
* If you are using Vertex AI for chat, text generation, or other non-embedding use cases, you must migrate to an alternative model provider. The <code>spring-ai-vertex-ai-embedding</code> module for embeddings is still available. Remove any dependencies on <code>spring-ai-vertex-ai-gemini</code> or related non-embedding autoconfiguration modules from your project. <a href="https://github.com/spring-projects/spring-ai/pull/5714" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5714/hovercard">#5714</a>
* If your project uses the ZhipuAI model or its auto-configuration from Spring AI, you must migrate to an alternative. The integration has been removed from the main repository as of this release. <a href="https://github.com/spring-projects/spring-ai/pull/5700" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5700/hovercard">#5700</a>
* If your project uses OCI GenAI support from Spring AI, you must migrate to an alternative. The integration has been removed from the main repository. Users relying on OCI GenAI will be able to migrate to a separate integration that will be released and documented shortly. <a href="https://github.com/spring-projects/spring-ai/pull/5695" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5695/hovercard">#5695</a>
* Projects using <code>SpringAiTestAutoConfigurations</code> for test setup must update their test configuration. Review your test context setup and replace any usage of this removed class with appropriate alternatives. <a href="https://github.com/spring-projects/spring-ai/pull/5684" data-hovercard-type="pull_request" data-hovercard-url="/spring-projects/spring-ai/pull/5684/hovercard">#5684</a>
