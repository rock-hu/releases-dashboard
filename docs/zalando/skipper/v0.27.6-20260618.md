feature: Add support to opt-out from zone aware routing via Ingress/RouteGroup (<a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="4589161775" data-permission-text="Title is private" data-url="https://github.com/zalando/skipper/issues/4049" data-hovercard-type="pull_request" data-hovercard-url="/zalando/skipper/pull/4049/hovercard" href="https://github.com/zalando/skipper/pull/4049">#4049</a>)

Introduce the annotation <code>zalando.org/traffic-zone-aware: "false"</code> on  

Ingress and RouteGroup resources. When set, the resource receives all  

endpoints across all zones, bypassing zone filtering both at the  

dataclient layer (endpoint fetch) and at the routesrv layer (per-zone  

route serving).

Usage:

    metadata:
      annotations:
        zalando.org/traffic-zone-aware: "false"

Tasks

1. Define <code>zalando.org/traffic-zone-aware</code> annotation constant in the  
   kubernetes package
2. Modify <code>GetEndpointSlicesByService</code> and <code>GetEndpointSlicesByTarget</code>  
   in <code>clusterstate.go</code> to accept ing and rg context param respectively and  
   skip zone filtering when value  
   ing.disableZoneAwareness/rg.disableZoneAwareness is true
3. Add zoneAwarenessAnnotationFilter() helper that returns an annotate()  
   filter when the zone-aware opt-out annotation is present
4. Modify Ingress v1 path rule conversion (<code>convertPathRuleV1</code>) and  
   default backend conversion (<code>convertDefaultBackendV1</code>) to read the  
   annotation, pass it to endpoint fetch and prepend the annotate() filter  
   when annotation is set
5. Modify RouteGroup backend application (<code>applyServiceBackend</code>) to read  
   the annotation, pass it to endpoint fetch and prepend the annotate()  
   filter when annotation is set
6. Add isZoneAwarenessDisabled() in routesrv/polling.go to detect the  
   opt-out by inspecting the route filter chain
7. Wire isZoneAwarenessDisabled() into getRouteForZone() to skip zone  
   filtering for opted-out routes
8. Update makeRoute test helper to inject the annotate() filter when the  
   opt-out flag is passed
9. Add integration test with two Ingresses in zone <code>eu-central-1a</code>: one  
   opted out (expects all endpoints across zones), one default (expects  
   only zone-local endpoints)
10. Add unit tests for <code>filterRoutesByZone</code> covering opt-out scenarios:  
    both routes opted out, opt-out mixed with normal route, and opt-out with  
    a below-threshold zone
