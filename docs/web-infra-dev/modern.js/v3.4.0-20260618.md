更新内容

新特性 🎉

* feat(server-core): 为 server 插件 fallback hook 传递 request 和 monitors 上下文， 详情可查看 <a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="4647075167" data-permission-text="Title is private" data-url="https://github.com/web-infra-dev/modern.js/issues/8678" data-hovercard-type="pull_request" data-hovercard-url="/web-infra-dev/modern.js/pull/8678/hovercard" href="https://github.com/web-infra-dev/modern.js/pull/8678">#8678</a>
* <code>server.rsc</code> now accepts an object form <code>{ environments?: { server?: string; client?: string } }</code> in addition to a boolean. This forwards the existing <code>environments</code> option of <code>rsbuild-plugin-rsc</code>, letting frameworks that declare their own Rsbuild environments map RSC onto them instead of having the plugin create new empty <code>server</code>/<code>client</code> environments (which otherwise fall back to the default <code>./src</code> entry and fail to resolve in non-convention setups). Passing <code>true</code>/<code>false</code> keeps the previous default behavior, so the change is fully backward compatible.  
  feat(builder): 支持通过 <code>server.rsc.environments</code> 自定义 RSC server/client 环境名  
  <code>server.rsc</code> 在原有 boolean 之外新增对象形式 <code>{ environments?: { server?: string; client?: string } }</code>，透传 <code>rsbuild-plugin-rsc</code> 已有的 <code>environments</code> 选项，使已声明自有 Rsbuild 环境的框架可将 RSC 映射到这些环境，而不必让插件新建空的 <code>server</code>/<code>client</code> 环境（否则会回落到默认入口 <code>./src</code> 而无法解析）。传 <code>true</code>/<code>false</code> 时行为不变，完全向后兼容。， 详情可查看 <a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="4662873511" data-permission-text="Title is private" data-url="https://github.com/web-infra-dev/modern.js/issues/8691" data-hovercard-type="pull_request" data-hovercard-url="/web-infra-dev/modern.js/pull/8691/hovercard" href="https://github.com/web-infra-dev/modern.js/pull/8691">#8691</a>
