Highlights üí°

üéà Upgrade SWC

The latest version of <a href="https://swc.rs/docs/plugin/ecmascript/compatibility" rel="nofollow">SWC enhances the compatibility of Wasm plugins</a>. This means future upgrades of Rspack and SWC will almost no longer break SWC Wasm plugins. Additionally, the SWC upgrade brings a 10% performance improvement to the parser.

‚úÖ Enable lazy compilation by default

Since Rspack v1.5 has stabilized <a href="https://rspack.rs/guide/features/lazy-compilation" rel="nofollow">Lazy Compilation</a>, we are now enabling it by default for dynamic imports cases when the <a href="https://rspack.rs/config/target#target" rel="nofollow">target</a> is <code>web</code>.

    export default defineConfig({
      lazyCompilation: {
        imports: true,
      },
    });

üéØ Stablize several experimental configurations

In the previous releases, the experimental features <a href="https://rspack.rs/config/experiments#experimentslazybarrel" rel="nofollow">experiments.lazyBarrel</a>, <a href="https://rspack.rs/config/experiments#experimentsinlineconst" rel="nofollow">experiments.inlineConst</a>, <a href="https://rspack.rs/config/experiments#experimentsinlineenum" rel="nofollow">experiments.inlineEnum</a> were introduced to enable cross-module inlining optimizations for constants. These optimizations improve minifiers' ability to perform precise static analysis, eliminate unused code branches, and further reduce bundle size.

After thorough validation, we are now promoting these features from experimental to stable.

    export default  {
      experiments: {
    -   inlineConst: true,
    -   inlineEnum: true,
    -   lazyBarrel: true,
    -   typeReexportsPresence: true,
      },
      module: {
        parser: {
          javascript: {
    -      inlineConst: true,
           typeReexportsPresence: true,
          },
        },
      },
      optimization: {
    +  inlineExports: true,
      },
    }
