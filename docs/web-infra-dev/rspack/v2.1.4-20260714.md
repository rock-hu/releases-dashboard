Highlights

Fine-grained <code>import.meta</code> parser options

<code>module.parser.javascript.importMeta</code> now accepts an object so each known <code>import.meta</code> property can be configured independently, on top of the existing <code>true</code> / <code>false</code> / <code>'preserve-unknown'</code> values. Properties set to <code>false</code> are preserved for runtime evaluation, omitted properties keep their default behavior, and unknown properties are preserved. Both webpack-compatible keys (<code>dirname</code>, <code>filename</code>, <code>main</code>, <code>url</code>, <code>webpack</code>, <code>webpackContext</code>) and Rspack-specific keys (<code>rspackHash</code>, <code>rspackPublicPath</code>, <code>glob</code>, ...) are supported.

    export default {
      module: {
        parser: {
          javascript: {
            importMeta: {
              // keep `import.meta.url` for runtime evaluation
              url: false,
              // keep the compile-time replacement of `import.meta.webpack`
              webpack: true,
            },
          },
        },
      },
    };

Static worker URL output

<code>module.parser.javascript.worker</code> now accepts an object form with <code>alias</code> and <code>url</code>, alongside the existing <code>boolean</code> and <code>string[]</code> forms. With <code>url: 'new-url-relative'</code> and <code>output.module</code> enabled, Rspack emits a static <code>new URL()</code> expression for <code>new Worker(new URL(..., import.meta.url))</code> instead of runtime code, honoring <code>output.publicPath</code> and <code>output.workerPublicPath</code>.

    export default {
      output: {
        module: true,
        chunkFilename: '[name].bundle.js',
      },
      module: {
        parser: {
          javascript: {
            worker: {
              url: 'new-url-relative',
            },
          },
        },
      },
    };

    new Worker(new URL('./worker.js', import.meta.url));

    // would become 👇
    new Worker(new URL('./worker.bundle.js', import.meta.url));
