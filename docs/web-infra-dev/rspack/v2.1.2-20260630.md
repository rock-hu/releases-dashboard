What's Changed

Highlights 💡

Support <code>import.meta</code> style module variables

Rspack now exposes more module variables through <code>import.meta</code>, which aligns better with the ESM specification than CommonJS-style injected globals. When authoring ESM modules, <code>import.meta</code> is now the recommended way to access these runtime values.

The newly supported <code>import.meta</code> properties are:

| New import.meta API | Existing runtime variable |
|--------------------------------------------|---------------------------------------|
| <code>import.meta.rspackPublicPath</code> | <code>__webpack_public_path__</code> |
| <code>import.meta.rspackBaseUri</code> | <code>__webpack_base_uri__</code> |
| <code>import.meta.rspackShareScopes</code> | <code>__webpack_share_scopes__</code> |
| <code>import.meta.rspackInitSharing</code> | <code>__webpack_init_sharing__</code> |
| <code>import.meta.rspackNonce</code> | <code>__webpack_nonce__</code> |
| <code>import.meta.rspackUniqueId</code> | <code>__rspack_unique_id__</code> |
| <code>import.meta.rspackVersion</code> | <code>__rspack_version__</code> |
| <code>import.meta.rspackHash</code> | <code>__webpack_hash__</code> |
