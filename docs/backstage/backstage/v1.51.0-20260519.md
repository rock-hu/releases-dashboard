# v1.51.0

## release on 20260519
## description
## changes
These are the release notes for the v1.51.0 release of <a href="https://backstage.io/" rel="nofollow">Backstage</a>.

A huge thanks to the whole team of maintainers and contributors as well as the amazing Backstage Community for the hard work in getting this release developed and done.

Highlights

<strong>BREAKING</strong>: Removed deprecated <code>NavItemBlueprint</code>

The deprecated <code>NavItemBlueprint</code> has been removed from <code>@backstage/frontend-plugin-api</code>. Navigation items are now discovered from <code>PageBlueprint</code> extensions based on their <code>title</code> and <code>icon</code> params. If you were still using <code>NavItemBlueprint</code>, migrate by setting <code>title</code> and <code>icon</code> on the page extension instead. All built-in plugins have been updated accordingly.

Additionally, <code>renderInTestApp</code> from <code>@backstage/frontend-test-utils</code> no longer renders a sidebar or legacy <code>nav-item</code> extensions. The app nav extension is now disabled in the minimal test app shell.

<strong>BREAKING</strong>: Removed deprecated <code>PortableSchema.schema</code> property form

The deprecated property form of <code>PortableSchema.schema</code> has been removed from <code>@backstage/frontend-plugin-api</code>. The <code>schema</code> member is now a plain method that must be called as <code>schema()</code> — direct property access like <code>schema.type</code> or <code>schema.properties</code> is no longer supported.

<strong>BREAKING</strong>: Hardened OIDC default patterns

The default allowed patterns for CIMD and DCR in <code>@backstage/plugin-auth-backend</code> have been hardened. The previous permissive <code>['*']</code> wildcards have been replaced with specific defaults for known MCP clients. If you previously relied on the permissive defaults and have custom MCP clients, you will need to explicitly add their patterns to the allow list.

<strong>BREAKING</strong>: Cleaned up <code>PolicyQueryUser</code> type

The <code>token</code> and <code>expiresInSeconds</code> fields have been removed from <code>PolicyQueryUser</code> in <code>@backstage/plugin-permission-node</code>. These were previously deprecated in favor of <code>credentials</code> with <code>coreServices.auth</code>. The <code>identity</code> field has been deprecated. A new <code>CachedUserInfoService</code> with a 5-second TTL cache and in-flight request coalescing has been added to reduce repeated user info lookups.

<strong>BREAKING</strong>: Catalog entity pagination excludes entities without sort field

When paginating entities with an order field via <code>/entities/by-query</code>, entities that lack the order field are now excluded from both the result set and the <code>totalItems</code> count. Previously these entities appeared at the end via <code>NULLS LAST</code>, but cursor-based pagination could not actually reach them past the first page — the count over-reported the number of navigable entities.

<strong>BREAKING</strong>: Microsoft Graph disabled users filtered by default

The <code>@backstage/plugin-catalog-backend-module-msgraph</code> and <code>@backstage/plugin-catalog-backend-module-msgraph-incremental</code> providers now filter out disabled user accounts by default. The provider automatically applies an <code>accountEnabled eq true</code> filter, combining it with any custom <code>user.filter</code> you provide. If you need to ingest disabled accounts, set the filter to explicitly include them.

Contributed by <a href="https://github.com/mtlewis">@mtlewis</a> in <a href="https://github.com/backstage/backstage/pull/34165" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/34165/hovercard">#34165</a>

<strong>BREAKING</strong>: Backstage UI updates

There are several new additions in Backstage UI:

<strong>New components:</strong> A <code>Combobox</code> component pairs a text input with a filterable dropdown, supporting sectioned options, icons, sizes, and custom typed values. New <code>DatePicker</code> and <code>DateRangePicker</code> components provide accessible date selection with calendar popovers built on React Aria. Flex item props (<code>grow</code>, <code>shrink</code>, <code>basis</code>) have been added to <code>Box</code>, <code>Card</code>, <code>Grid</code>, and <code>Flex</code>.

<strong>Header improvements:</strong> A <code>sticky</code> prop has been added to the <code>Header</code> component that keeps the title-and-actions bar fixed at the top of its scroll container. New <code>description</code>, <code>tags</code>, and <code>metadata</code> props provide richer header content. The <code>breadcrumbs</code> prop has been deprecated.

<strong>Other additions:</strong> Grouped options in <code>Select</code>, <code>isPending</code> prop replacing <code>loading</code> across components, <code>searchDebounceMs</code> and <code>filterDebounceMs</code> options for <code>useTable</code>, <code>PasswordField</code> visual alignment with <code>TextField</code>, a public <code>--bui-bg-inherit</code> CSS variable, and keyboard focus indicators on <code>Card</code> links.

<strong>Breaking changes:</strong>

* <strong>Header</strong>: Removed the main header class from the <code>Header</code> component. Custom styles targeting this class should be updated.
* <code>@remixicon/react</code> dependency limited to versions below 4.9.0 due to a license change.
* React Aria dependencies updated to v1.17.0 and migrated to monopackages.

The <code>DatePicker</code> was contributed by <a href="https://github.com/Swiftwork">@Swiftwork</a> in <a href="https://github.com/backstage/backstage/pull/34184" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/34184/hovercard">#34184</a>. Check the <a href="https://ui.backstage.io/changelog" rel="nofollow">BUI Changelog</a> for more details.

<code>AiResource</code> catalog entity kind

A new <code>AiResource</code> catalog entity kind has been introduced, with entity types, validators, type guards, and model layer definitions exported from <code>@backstage/catalog-model/alpha</code>. Install <code>@backstage/plugin-catalog-backend-module-ai-model</code> to enable it. A new <code>spec.type: 'mcp-server'</code> structured subtype has also been added to the <code>API</code> kind, with a <code>spec.remotes</code> list for representing MCP server connections.

New plugin: Microsoft Graph incremental ingestion

A new <code>@backstage/plugin-catalog-backend-module-msgraph-incremental</code> module provides cursor-based incremental ingestion for Microsoft Graph. Unlike <code>MicrosoftGraphOrgEntityProvider</code>, this module never holds the full dataset in memory — each burst processes a single page of up to 999 users or 100 groups. The cursor is persisted so a pod restart resumes from the last completed page.

Contributed by <a href="https://github.com/sriharsha9618">@sriharsha9618</a> in <a href="https://github.com/backstage/backstage/pull/34053" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/34053/hovercard">#34053</a>

Scaffolder form decorators promoted to stable

The <code>formDecorators</code> field on template specs, the <code>formDecoratorsApiRef</code> API, and the <code>FormDecoratorBlueprint</code> have been promoted from <code>@alpha</code> to <code>@public</code> across <code>@backstage/plugin-scaffolder</code>, <code>@backstage/plugin-scaffolder-react</code>, <code>@backstage/plugin-scaffolder-common</code>, and <code>@backstage/plugin-scaffolder-backend</code>. The previous <code>EXPERIMENTAL_formDecorators</code> field continues to work as a deprecated alias. Decorator input is now validated against the configured zod schema before execution.

Experimental BUI scaffolder form theme

An experimental Backstage UI form theme has been added for scaffolder forms. All default field extensions render BUI variants when enabled. Set <code>enableBackstageUi: true</code> in the <code>sub-page:scaffolder/templates</code> extension config to try it out.

Template groups configuration

The <code>sub-page:scaffolder/templates</code> extension now accepts a <code>groups</code> config field for defining template groups on the template list page. Each group has a <code>title</code> and a <code>filter</code> predicate. Templates not matched by any group fall into an "Other Templates" group.

    app:
      extensions:
        - sub-page:scaffolder/templates:
            config:
              groups:
                - title: Recommended Services
                  filter:
                    spec.type: service
                - title: Documentation
                  filter:
                    spec.type: documentation

The <code>TemplateCard</code> is now a swappable component — apps using the new frontend system can replace it via <code>SwappableComponentBlueprint</code>.

<code>ExtensionPointFactoryMiddleware</code> for backend

A new <code>ExtensionPointFactoryMiddleware</code> type and <code>createExtensionPointFactoryMiddleware</code> helper have been added to <code>@backstage/backend-app-api</code>, allowing extension point outputs to be replaced at backend creation time. The <code>defaultServiceFactories</code> export has been added to <code>@backstage/backend-defaults</code> for use with <code>createSpecializedBackend</code>.

Contributed by <a href="https://github.com/UsainBloot">@UsainBloot</a> in <a href="https://github.com/backstage/backstage/pull/33782" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/33782/hovercard">#33782</a>

Alpha <code>TracingService</code>

An alpha <code>TracingService</code> has been added to <code>@backstage/backend-plugin-api</code> and <code>@backstage/backend-defaults</code>, providing a unified interface for emitting trace spans across Backstage plugins. The service includes <code>context</code> and <code>propagation</code> support for bridging OpenTelemetry context across async boundaries. MCP <code>tools/call</code> invocations now emit trace spans following OpenTelemetry server-side MCP semantic conventions. A corresponding mock is available in <code>@backstage/backend-test-utils</code>.

Contributed by <a href="https://github.com/iamEAP">@iamEAP</a> in <a href="https://github.com/backstage/backstage/pull/34087" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/34087/hovercard">#34087</a>

Catalog performance improvements

Several performance improvements have been made to the catalog backend:

* The entity listing endpoint now lets PostgreSQL walk the <code>(key, value, entity_id)</code> index in sorted order and short-circuit on <code>LIMIT</code>, reducing typical paginated list times from seconds to milliseconds.
* The entity facets endpoint uses an inner join rather than <code>WHERE entity_id IN (subquery)</code> when filters are applied, with measured improvements from ~1.2x to 7x.
* The facets aggregation simplified from <code>COUNT(DISTINCT entity_id)</code> to <code>COUNT(*)</code>, enabled by the new unique constraint.
* The <code>catalog_entities_count</code> metric now shares a single cached query between Prometheus and OpenTelemetry gauges.
* A missing index on <code>relations.target_entity_ref</code> has been added, fixing full sequential scans on orphan deletion, entity ancestry, and eager pruning queries.
* Incremental ingestion <code>WHERE ref IN (...)</code> queries now use <code>= ANY($1)</code> with a single array parameter to reduce prepared statement bloat.
* A new migration removes duplicate rows from the <code>search</code> table, creates covering indices, and adds a <code>UNIQUE</code> constraint on <code>(entity_id, key, value)</code>. <strong>For large installations</strong>, it is recommended to run the provided SQL commands before deploying — see the changelog for details.

New ESLint <code>no-self-package-imports</code> rule

A new <code>no-self-package-imports</code> lint rule has been added to <code>@backstage/eslint-plugin</code>, enabled as <code>error</code> in the recommended config. It reports when a package imports itself by its own name instead of using a relative path — a pattern that causes circular initialization errors in bundled ESM and with <code>jest.requireActual</code>.

Contributed by <a href="https://github.com/dyatko">@dyatko</a> in <a href="https://github.com/backstage/backstage/pull/34041" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/34041/hovercard">#34041</a>

AWS web identity token file support

Added <code>webIdentityTokenFile</code> to <code>@backstage/integration-aws-node</code> account configuration. When set along with a <code>roleName</code>, <code>DefaultAwsCredentialsManager</code> retrieves credentials by calling <code>AssumeRoleWithWebIdentity</code> using the file's contents as the web identity token. The file is re-read on each credential refresh.

Contributed by <a href="https://github.com/hudsonb">@hudsonb</a> in <a href="https://github.com/backstage/backstage/pull/34149" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/34149/hovercard">#34149</a>

TechDocs: disable external font downloads

Added support for disabling external font downloads in TechDocs, useful for air-gapped Backstage instances. Available via the <code>techdocs.generator.mkdocs.disableExternalFonts</code> app-config option and the <code>techdocs-cli generate --disableExternalFonts</code> CLI flag.

Contributed by <a href="https://github.com/karthikjeeyar">@karthikjeeyar</a> in <a href="https://github.com/backstage/backstage/pull/31838" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/31838/hovercard">#31838</a>

<code>always()</code> and <code>failure()</code> status functions for scaffolder steps

Added <code>always()</code> and <code>failure()</code> status check functions for scaffolder steps. These can be used in the <code>if</code> field to control execution after failures. <code>always()</code> ensures a step runs regardless of previous step outcomes, while <code>failure()</code> runs a step only when a previous step has failed.

Contributed by <a href="https://github.com/Ferin79">@Ferin79</a> in <a href="https://github.com/backstage/backstage/pull/32890" data-hovercard-type="pull_request" data-hovercard-url="/backstage/backstage/pull/32890/hovercard">#32890</a>

Deprecated immediate mode stitching

The <code>catalog.stitchingStrategy.mode: 'immediate'</code> setting has been deprecated. A warning is now logged on startup when immediate mode is configured. Immediate mode will be removed in the next Backstage release.
