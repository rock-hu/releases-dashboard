Added

* Added reference to Open License in README.md (<a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="3457883106" data-permission-text="Title is private" data-url="https://github.com/OpenBankingUK/read-write-api-specs/issues/191" data-hovercard-type="pull_request" data-hovercard-url="/OpenBankingUK/read-write-api-specs/pull/191/hovercard" href="https://github.com/OpenBankingUK/read-write-api-specs/pull/191">#191</a>).
* Added <code>DCSV</code> to <code>OBFileFormat</code> schema in account-info-openapi. This addresses v40_KI42

Changed

* v40_KI34 - Update regex for <code>LEI</code> in PIS & VRP
  * Addresses v40_KI34 which aligns the PIS regex for <code>LEI</code> to the spec pages and updates the VRP swagger to include the regex & example.
* Updated Payment Status enums to match Spec pages.
  * Some payment status codes were present on the Spec pages but not in the swagger.

* Fixed VRP PATCH operation description and schema.
  * The description was appearing in the <code>format</code> field and has been moved to the correct <code>description</code> field. Additonally the request body schemea has been updated to fully reflect <a href="https://datatracker.ietf.org/doc/html/rfc6902" rel="nofollow">RFC 6902</a>.
* Corrected a regex typo in AIS <code>OB_CodeMnemonic</code> (<a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="3260262420" data-permission-text="Title is private" data-url="https://github.com/OpenBankingUK/read-write-api-specs/issues/189" data-hovercard-type="pull_request" data-hovercard-url="/OpenBankingUK/read-write-api-specs/pull/189/hovercard" href="https://github.com/OpenBankingUK/read-write-api-specs/pull/189">#189</a>). Related to v40_KI30.
* Restored <code>Multiple</code> to instances of <code>OBInternalAuthorisation1Code</code>. Addresses Known Issue v40_KI44
