# Version 2.24.0

## release on 20260117
## description
## changes
This release targets the OpenTelemetry SDK 1.58.0.

Note that many artifacts have the <code>-alpha</code> suffix attached to their version number, reflecting that they are still alpha quality and will continue to have breaking changes. Please see the <a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/blob/main/VERSIONING.md#opentelemetry-java-instrumentation-versioning">VERSIONING.md</a> for more details.

:warning: Breaking Changes

* Remove support for previously deprecated property <code>otel.instrumentation.logback-appender.experimental.capture-logstash-attributes</code> (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15722" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15722/hovercard">#15722</a>)
* Remove deprecated methods from HTTP library instrumentations (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15802" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15802/hovercard">#15802</a>)

üö´ Deprecations

* JMX Metrics: Deprecated <code>addClassPathRules</code> and <code>addCustomRules</code> methods in <code>JmxTelemetryBuilder</code>, and moved non-public classes to an internal package (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15658" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15658/hovercard">#15658</a>)
* Deprecated <code>setMessagingReceiveInstrumentationEnabled</code> in favor of <code>setMessagingReceiveTelemetryEnabled</code> to match config property name (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15668" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15668/hovercard">#15668</a>)
* Deprecated database client attribute getter methods <code>getDbSystem()</code> and <code>getResponseStatus()</code> in favor of <code>getDbSystemName()</code> and <code>getResponseStatusCode()</code> to align with stable semantic conventions (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15696" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15696/hovercard">#15696</a>)
* Deprecated <code>setCapturedRequestParameters()</code> in Servlet library instrumentation in favor of <code>Experimental.setCapturedRequestParameters()</code> as request parameter capture is experimental (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15826" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15826/hovercard">#15826</a>)
* Deprecated <code>getHttpClient()</code>, <code>setHttpClientTransport()</code>, and <code>setSslContextFactory()</code> in Jetty client instrumentation in favor of new builder-style methods <code>newHttpClient()</code>, <code>newHttpClient(SslContextFactory)</code>, and <code>newHttpClient(HttpClientTransport, SslContextFactory)</code> (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15827" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15827/hovercard">#15827</a>)
* Deprecate Netty experimental method that is still in public API (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15828" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15828/hovercard">#15828</a>)
* Deprecated <code>newHttpClient()</code> in favor of <code>wrap()</code> in Java HTTP Client instrumentation for consistency with naming conventions across library instrumentations (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15829" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15829/hovercard">#15829</a>)
* Deprecated <code>new*</code> methods in favor of <code>create*</code> methods across multiple instrumentation libraries for consistency (e.g., <code>newHttpClient()</code> ‚Üí <code>createHttpClient()</code>, <code>newInterceptor()</code> ‚Üí <code>createInterceptor()</code>) (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15832" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15832/hovercard">#15832</a>)
* Deprecated methods in <code>SqlStatementInfo</code> and <code>MultiQuery</code> in favor of methods using stable semantic convention terminology: <code>getFullStatement()</code> ‚Üí <code>getQueryText()</code>, <code>getOperation()</code> ‚Üí <code>getOperationName()</code>, and <code>getStatements()</code> ‚Üí <code>getQueryTexts()</code> (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15833" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15833/hovercard">#15833</a>)
* Deprecated database getter methods in favor of methods using stable semantic convention terminology: <code>getResponseStatusCode()</code> ‚Üí <code>getDbResponseStatusCode()</code>, <code>getBatchSize()</code> ‚Üí <code>getDbOperationBatchSize()</code>, and <code>getQueryParameters()</code> ‚Üí <code>getDbQueryParameters()</code> (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15859" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15859/hovercard">#15859</a>)
