# Version 2.29.0

## release on 20260619
## description
## changes
This release targets the OpenTelemetry SDK 1.63.0.

Note that many artifacts have the <code>-alpha</code> suffix attached to their version number, reflecting that they will continue to have breaking changes. Please see <a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/blob/main/VERSIONING.md#opentelemetry-java-instrumentation-versioning">VERSIONING.md</a> for more details.

:warning: Breaking changes to non-stable APIs

* Change the return type for <code>JmxTelemetry.start(...)</code> APIs. (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/18782" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/18782/hovercard">#18782</a>)
* Experimental runtime package telemetry now emits <code>package.checksum_algorithm=SHA-256</code> and 64-character <code>package.checksum</code> values instead of SHA1 checksums. (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/18846" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/18846/hovercard">#18846</a>)

🚫 Deprecations

* Rename the common logging context keys to <code>otel.instrumentation.common.logging.trace-id-key</code>, <code>otel.instrumentation.common.logging.span-id-key</code>, and <code>otel.instrumentation.common.logging.trace-flags-key</code>, while keeping the old property names deprecated. (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/18851" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/18851/hovercard">#18851</a>)
* Rename the <code>kafka-broker</code> and <code>kafka-connect</code> JMX target systems to <code>experimental-kafka-broker</code> and <code>experimental-kafka-connect</code>; the old names still work for now and log a warning. (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/18971" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/18971/hovercard">#18971</a>)

🌟 New javaagent instrumentation

* Add SOFARPC 5.4 instrumentation that emits RPC client and server spans and metrics. (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/15589" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/15589/hovercard">#15589</a>)
* Add Javaagent instrumentation for HBase client 2.0.0 through 2.5.0. (<a href="https://github.com/open-telemetry/opentelemetry-java-instrumentation/pull/18253" data-hovercard-type="pull_request" data-hovercard-url="/open-telemetry/opentelemetry-java-instrumentation/pull/18253/hovercard">#18253</a>)
