# SRU2025-9.6.7

## release on 20260407
## description
## changes
9.6.7 - April 2026

* Fix: Replaced <code>@OrderColumn</code> with <code>@OrderBy("creationDate ASC")</code> on <code>statusTrail</code>, <code>notes</code>, and <code>revisions</code> to prevent data loss caused by <code>sort_key</code> corruption under concurrent access. Existing <code>sort_key</code> columns must be made nullable or dropped.
* Feat: Improved performance of <code>SwiftParseUtils.getLines()</code> by replacing <code>BufferedReader</code> with direct string index parsing
* Feat: <code>MtSwiftMessage</code> created from ACK (service 21) followed by MT with block 2 Output now extracts the message type and metadata from the appended MT instead of defaulting to ACK
* Migrated deprecated <code>StringUtils</code> methods to <code>Strings.CS</code> equivalents (equals, startsWith, endsWith, contains, replace, indexOf, lastIndexOf, remove, removeEnd)
* (PW-3126) Fixed DN to BIC extraction in <code>DistinguishedName</code> to include the branch code from the <code>ou</code> component
* (PW-3123) Feat: Added <code>checksum()</code> and <code>checksumBody()</code> default methods to <code>MessageMetadataStrategy</code> (useful for duplicate detection)
* (PW-3123) Feat: Made <code>SwiftMessageUtils.md5()</code> public for checksum computation in custom implementations
* Deprecated <code>Field.validatorPattern()</code> - validation patterns moved to Integrator's <code>FieldPatternRegistry</code>
* Updated MT message structures from schema regeneration (MT081, MT513-MT543, MT586, MT920, MT942)
* (PW-2967) Fixed Field95D component label from "Legal Entity Identifier" to "Digital Ledger Identifier" to match XSD schema
* Feat: <code>MtSwiftMessage</code> now automatically extracts and persists UETR from block 3 field 121 during message parsing
* Fix: Added missing <code>uetr</code> field to <code>AbstractSwiftMessage</code> <code>copyTo()</code>, <code>equals()</code>, and <code>hashCode()</code> methods
