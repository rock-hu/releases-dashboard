# SRU2025-9.6.8

## release on 20260702
## description
## changes
9.6.8 - July 2026

* (PW-3185) Fix: <code>MtSwiftMessage.toJson()</code> now uses 1-based months (January=1) for Calendar fields and emits a <code>schemaVersion</code> marker; <code>fromJson()</code> reads both new and legacy (0-based) payloads transparently
* Fix: <code>SwiftTagListBlock.getFieldsByNumber(int)</code> no longer throws <code>IllegalArgumentException</code> when a matching tag cannot be converted into a <code>Field</code>; such tags are now logged and skipped, consistently with the name-based lookups
* Fix: Fields definitions alignment: <code>Field30I</code> and <code>Field30K</code> component 2 made optional (and <code>Field30K</code> validation pattern updated to <code>&lt;DATE4&gt;[/&lt;DATE4&gt;]</code>), <code>Field39M</code> validation pattern updated to <code>&lt;CC&gt;</code>, and <code>Field56B</code> name & address component made optional
* Fix: Minor changes in MT message structures from SRU2025 schema: MT306, MT340, MT500-MT502, MT504, MT508, MT510, MT513-MT515, MT518, MT519, MT524, MT527, MT530, MT535-MT538, MT540-MT548, MT558, MT564-MT566, MT575, MT578, MT586, MT600, MT601, MT671
* Feat: Completed and consolidated the field labels across all language bundles (de, en, es, fr, it, ru), filling in missing translations and replacing redundant message-specific entries with generic field names
