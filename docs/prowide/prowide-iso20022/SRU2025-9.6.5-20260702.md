# SRU2025-9.6.5

## release on 20260702
## description
## changes
9.6.5 - July 2026

* (PW-3251) Feat: lenient parsing of file-format (FileAct) payloads with sibling <code>AppHdr</code> and <code>Document</code> root elements or undeclared namespace prefixes, applied consistently across all parsing entry points without copying the payload
* (PW-3251) <code>MxParseUtils.identifyMessage</code> now returns an empty Optional on blank input instead of throwing an IllegalArgumentException
* Feat: <code>xsd:any</code> wildcard content (e.g. <code>SplmtryData/Envlp</code>, signature envelopes) is no longer dropped on parse; it is captured as an <code>org.w3c.dom.Element</code> with namespaces preserved (<a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="1100153671" data-permission-text="Title is private" data-url="https://github.com/prowide/prowide-iso20022/issues/39" data-hovercard-type="issue" data-hovercard-url="/prowide/prowide-iso20022/issues/39/hovercard" href="https://github.com/prowide/prowide-iso20022/issues/39">GH-39</a>, <a class="issue-link js-issue-link" data-error-text="Failed to load title" data-id="1161454200" data-permission-text="Title is private" data-url="https://github.com/prowide/prowide-iso20022/issues/43" data-hovercard-type="issue" data-hovercard-url="/prowide/prowide-iso20022/issues/43/hovercard" href="https://github.com/prowide/prowide-iso20022/issues/43">GH-43</a>)
* Fix: <code>toJson()</code> now serializes <code>@XmlAnyElement</code> wildcard content as raw XML instead of an empty <code>{}</code>, and <code>fromJson()</code> restores it back to a DOM Element (round-trippable)
* (PW-3202) Fix: removed spurious <code>Error propagating pending prefix mapping</code> warnings when parsing regular ISO 20022 Documents wrapped in an SNL-like envelope
* (PW-3185) Fix: <code>MxSwiftMessage.fromJson()</code> now reads Calendar fields in both the new 1-based month format (identified by a <code>schemaVersion</code> marker) and the legacy 0-based format transparently
* (PW-3207) Fix: update semt.044.001.01 model from draft4 to draft5
