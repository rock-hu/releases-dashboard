Spring Cloud Open Service Broker 4.5.2

Spring Cloud Open Service Broker 4.5.2 is a patch release in the 4.5.x line. Version 4.5.2 is built against Spring Boot 3.5, and requires Java 17+. It includes support for OSB API v2.16. This release reduces third-party library dependencies in favor of Spring-provided utilities, includes bug fixes, and upgrades core dependencies.

Highlights

* Spring Boot 3.5.16 (upgraded from Spring Boot 3.5.9)
* Gradle 9.5.1 build infrastructure
* Replace third-party commons-beanutils and commons-collections with Spring equivalents
* Bug fixes for log formatting and null safety in <code>ServiceInstanceBindingController</code>

Upgrade Notes

This release removes usage of Apache Commons libraries in favor of Spring Framework equivalents:

* <strong><code>commons-beanutils</code></strong> is no longer a dependency of <code>spring-cloud-open-service-broker-core</code>. It was previously pulled in as a direct dependency; <code>ParameterBeanMapperUtils</code> now uses Spring's <code>BeanWrapper</code> instead of <code>BeanUtilsBean</code>.

<strong>Action required:</strong> if your project relied on <code>commons-beanutils</code> being available on the classpath transitively through <code>spring-cloud-open-service-broker-core</code>, you will need to add it as an explicit dependency going forward.

Dependency Improvements

Remove Apache Commons Dependencies

* Replaced usage of commons-collections with Spring's <code>CollectionUtils</code> in <code>EventFlowRegistry</code>, eliminating the transitive dependency (<a href="https://github.com/spring-cloud/spring-cloud-open-service-broker/issues/971" data-hovercard-type="issue" data-hovercard-url="/spring-cloud/spring-cloud-open-service-broker/issues/971/hovercard">#971</a>).
* Replaced usage of commons-beanutils with Spring's <code>BeanWrapper</code> in <code>ParameterBeanMapperUtils</code>, eliminating the transitive dependency.

Build Dependency Cleanup

* Use the Spring Boot managed version of Hamcrest rather than a manually pinned version.
* Removed the Wiremock Gradle constraint, which was previously required for a Logback classpath conflict resolved by the Spring Cloud Contract upgrade.
* Reduced scope of the findbugs dependency from <code>testImplementation</code> to <code>testCompileOnly</code>.
* Removed the obsolete <code>junitPlatformLauncherVersion</code> variable.

Bug Fixes

* Fixed log formatting and output issues across <code>CatalogController</code>, <code>ServiceBrokerExceptionHandler</code>, <code>ServiceInstanceController</code>, and <code>ServiceInstanceBindingController</code> (<a href="https://github.com/spring-cloud/spring-cloud-open-service-broker/issues/945" data-hovercard-type="issue" data-hovercard-url="/spring-cloud/spring-cloud-open-service-broker/issues/945/hovercard">#945</a>).
* Fixed a null pointer risk in the operation equality check in <code>ServiceInstanceBindingController</code>, making the logic consistent with the equivalent check in <code>ServiceInstanceController</code>.

Improvements

* Simplified the service binding last operation response mapping in <code>ServiceInstanceBindingController</code>.
