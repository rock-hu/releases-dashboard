Spring Cloud Open Service Broker v5.0.0

Spring Cloud Open Service Broker 5.0.0 is now GA. Version 5.0.0 is built against Spring Boot 4.0, Spring Framework 7.0, Jackson 3, and requires Java 17+. It includes support for OSB API v2.16. This release includes significant upgrades to core dependencies and introduces null safety annotations across the API.

Highlights

* <strong>Spring Boot 4.0.3</strong> (upgraded from Spring Boot 3.5.x in the 4.x line)
* <strong>Jackson 3</strong> (<code>tools.jackson.core:jackson-databind</code>)
* <strong>Null safety with JSpecify</strong> across all public API packages
* <strong>Gradle 9.3.1</strong> build infrastructure (upgraded from Gradle 8.x)
* <strong>Java 25</strong> CI now verifies and tests against Java 17, 21, and 25

Breaking Changes

Spring Boot 4 Upgrade

* All Spring Boot dependencies and starters have been migrated to the 4.0.x line.
* Package, class, and method name changes from Spring Boot 4 are reflected throughout the codebase.

Jackson 3 Migration

* The core module now depends on <code>tools.jackson.core:jackson-databind</code> (Jackson 3) instead of <code>com.fasterxml.jackson.core:jackson-databind</code> (Jackson 2).
* Jackson 2 is retained only in test scope for JsonPath support.
* Model classes have been updated to apply Jackson 3 best practices:
  * <code>@JsonCreator</code> and <code>@JsonProperty</code> annotations added to model constructors.
  * Removed usage of <code>builderWithJackson2Defaults()</code>.
  * Reviewed and updated <code>@JsonProperty</code> usage on private members.

New Features

Null Safety with JSpecify

* Comprehensive null safety annotations have been added across all packages using <a href="https://jspecify.dev/" rel="nofollow">JSpecify</a>.
* <code>package-info.java</code> files added to all packages to declare null safety at the package level.
* JSpecify is declared as an <code>api</code> dependency, making null safety information available to consumers.
* This enables better static analysis and IDE support for service broker implementations.
