Spring Cloud Open Service Broker 5.1.0-M2

Spring Cloud Open Service Broker 5.1.0-M2 is the second milestone release in the 5.1.x line. This release focuses on Gradle build modernization, upgrading to Gradle 9.6.1 and removing several deprecated Gradle APIs and unnecessary plugins/configuration in preparation for future Gradle compatibility.

Versions

| Component | Version |
|------------------|---------|
| Spring Boot | 4.1.0 |
| Spring Framework | 7.0.8 |
| Java | 17+ |
| OSB API | 2.16 |

Highlights

* Upgrade to Gradle 9.6.1
* Remove deprecated and unnecessary Gradle build configuration
* Suppress spurious JDK warning during docs build

Upgrade Notes

The <code>spring-cloud-open-service-broker-webmvc</code> and <code>spring-cloud-open-service-broker-webflux</code> dependencies are no longer marked as optional in the published Maven POMs. Consumers that relied on this hint to see the dependency suggested transitively will no longer see it listed; this has no functional effect, and projects must continue to explicitly declare a Spring MVC or Spring WebFlux dependency as before.

Build Improvements

* Upgraded to Gradle 9.6.1.
* Replaced deprecated use of Gradle <code>getProperties</code>.
* Replaced use of Gradle's implicit parent property lookup.
* Fixed eager task configuration by using the modern Gradle <code>configureEach</code> block.
* Removed use of the deprecated <code>optionalApi</code> Gradle feature variant. Dependencies are no longer published as optional hints in the Maven POMs; implementing projects are still expected to include a Spring MVC or Spring WebFlux dependency.
* Removed unnecessary Gradle <code>eclipse</code> and <code>idea</code> plugins.
* Removed unnecessary <code>OUT_OF_PROCESS</code> execution mode from Gradle docs publication.
* Removed old Java version guards for Java 11 and 13, since the project requires Java 17+.
* Polished Gradle build files by removing unnecessary comments and whitespace.
* Pinned <code>spring-mock-mvc</code> to version 6.0.0 for consistency with the existing <code>rest-assured</code> pin, avoiding mixed Groovy 5 dependency versions.
* Suppressed a JDK IO subsystem warning from AsciidoctorJ's JRuby engine during the docs build.
* Bumped <code>actions/checkout</code> from 6 to 7.
